# Project workflow

For every user-requested update to this project:

1. Keep the divination output explicitly framed as symbolic reflection, not factual prediction.
2. Run `npm run check` after editing.
3. Commit the intended project files with a concise commit message.
4. Push the completed change to `origin/main` unless the user requests another branch workflow.
5. Confirm the Git-connected Vercel production deployment reaches `Ready`.
